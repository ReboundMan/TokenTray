# spec/

Canonical home for TokenUsageTray's specs, designs, and plans (ProjectPatterns Standard #1).
Every spec/plan markdown lives here — not at the repo root and not under `docs/`.

> Note: `TokenTray.spec` at the repo root is a **PyInstaller build spec**, not a project
> spec — it intentionally stays at the root next to the build tooling.

## Canonical files (ProjectPatterns standard)
These are the standardized artifacts each project should have. Add the missing ones as the
project matures:

| File | Purpose | Status |
|---|---|---|
| `SPEC.md` | One-page "what is this?" + metadata (slug, auth family, domain) | to add (#3) |
| `DEPLOY.md` | Deployment / release runbook | to add (#4) |
| `punchlist.md` | Feature / idea / bug capture (`punch "..."` appends here) | to add (#2) |
| `WORK_LOG.md` | Reverse-chronological log of meaningful work | to add (#5) |
| `card.json` | reboundman.com card metadata (read by `Sync-AppCards`) | to add (#6) |
